# Acrylic Pan 設計メモ

## 1. 推論問題

打撃の有無と位置推定は分けます。常時座標回帰を動かすのではなく、軽量な
しきい値処理で打撃イベントを切り出し、イベントごとにSolist-AIを1回実行します。
AIの初期出力は4 × 2領域に対応する8スコアです。スコア分布から期待座標を計算します。

```text
加速度Z → 打撃検出 → 1打分の特徴量 → Solist-AI → 8空間スコア
                                                    ├→ 最大値 → 音階
                                                    ├→ 確率 → ヒートマップ
                                                    └→ 期待座標(x, y)
```

8スコアを保持することで、最大値による領域判定だけでなく、曖昧さを含むヒートマップと
確率加重の連続座標を同じ出力から生成できます。直接座標回帰は実測点を増やした後に比較します。

## 2. ハードウェア前提

- パネル: 400 × 200 × 2 mm アクリル
- MCU/AI: ML63Q2557 + AxlCORE-ODL（DT-EBML63Q2557）
- センサ: KX134-1211、3軸、16 bit、別体MEMS基板を1個使用
- 初期サンプリング周波数: 6.4 kHz（付属アプリ標準設定）
- 前処理: 固定小数点からbfloat16への変換、FFTライブラリを利用可能

センサの固定条件は振動伝達そのものを変えるため、最終固定方法を決めてから本学習
データを収録します。パネル支持位置、ゴム足、ネジ締結、筐体との接触も同様に固定します。
センサ位置はパネル中央 `(200, 100) mm` とします。アクリルを挟み込む固定具により、
`x=200～300 mm、y=0～20 mm` の100 × 20 mm面を固定して境界条件を非対称化し、
左右鏡像点のモード強度差を作ります。

## 3. 教師データ

各サンプルには次を保存します。

- `session_id`: 収録日・組立状態を区別するID
- `hit_id`: 打撃イベントID
- `x_mm`, `y_mm`: 実際に叩いた座標
- `x_norm`, `y_norm`: Solist-AI教師値
- `force`: weak / normal / strong（人手ラベルでも可）
- `striker`: 指、マレットなど
- `accel_x`, `accel_y`, `accel_z`: 切り出した生波形

教師中心はX = 50, 150, 250, 350 mm、Y = 50, 150 mmです。中心だけに限定せず、
各100 mm領域の内側と境界付近にも打撃点を置きます。各領域80打程度を目安とし、
学習・検証・テストは打撃単位のランダム分割ではなく、収録セッション単位で分けます。

## 4. 特徴量候補

最初から1種類に固定せず、同一データで以下を比較します。

1. Z軸の符号付き時間波形: 128入力
2. 低次固有モード／FFT帯域強度: 18～64入力
3. Z軸FFT振幅
4. 時間波形にピーク時刻、減衰率を加えた特徴

打撃強度に依存しすぎないよう、イベント全体のRMSまたは最大値による正規化あり・なしを
比較します。ただし振幅自体にも位置情報があるため、完全正規化が常に有利とは限りません。

## 5. Solist-AIモデル

- 学習方式: 教師あり学習
- 出力数: 8
- ターゲット: 単打はone-hot、同時2点打撃はmulti-hotベクトル
- 損失: MSEを初期値とする
- 評価時に8スコアを確率化し、期待座標を計算
- 中間ノード: まず32、必要に応じて16/64/128を比較
- α: seed固定
- β: PC学習モデルとオンデバイス追加学習を比較

Solist-AI Simは連続数値の教師あり学習例を持つため、座標回帰と整合します。
ただし位置推定の成否はELM以前に、単一センサ波形へ位置固有情報がどれだけ残るかで
決まります。まずPython参照実験で可分性と距離誤差を確認します。

## 6. 評価指標

- X方向MAE [mm]
- Y方向MAE [mm]
- 2次元ユークリッド距離の平均・中央値・95パーセンタイル [mm]
- 4 × 2領域正解率
- 境界から20 mm以内／以外の領域正解率
- 打撃強度、打具、打撃者、セッション別の誤差
- 推論レイテンシ

50 mm領域への細分化は、独立テストで距離誤差が十分小さい場合だけ採用します。
目安として中央値15 mm以下、95パーセンタイル35 mm以下を最初の判断基準にします。

## 7. 音と表示

最初の実証ではMCUからUARTで推論座標と打撃強度をPCへ送り、PC側で低遅延に
サンプル音源を再生します。4 × 2領域には8音を割り当てます。打撃強度は音量へ反映します。

ヒートマップは最新推論点を明るく表示し、過去の点を時間減衰させて重ねます。推定点を
中心とする2次元Gaussianの幅には、検証データで得た座標誤差を使います。これにより
見た目だけでなく、モデルの実測精度を反映した表示になります。
