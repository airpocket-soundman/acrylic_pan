# CalculiX基準解析仕様

## 位置づけ

従来のKirchhoff有限差分モデルと独自HEX8 FEMは削除せず、設計探索・クロスチェック用として残す。提出資料の基準解析には、既存のオープンソースFEMソルバーCalculiX CrunchiXを使用する。

CalculiX公式サイト: https://www.calculix.de/

## ソフトウェア

| 項目 | 条件 |
|---|---|
| ソルバー | CalculiX CrunchiX 2.20、Debian `calculix-ccx=2.20-1` |
| 固有値内部解法 | SPOOLESによる分解、ARPACK shift-invert |
| コンテナベース | Debian bookworm-slim、digest `sha256:60eac759…511df` |
| 後処理 | Python 3.11、NumPy 1.24.2、SciPy 1.10.1、Matplotlib 3.6.3 |
| 動画・媒体 | FFmpeg 5.1.9（将来のCalculiX動画出力用） |
| 入力生成 | `calculix/generate_model.py` |
| 後処理 | `calculix/postprocess.py` |

## メッシュ設計

要素はC3D20R、20節点二次六面体・低減積分ソリッド要素とする。一次HEX8完全積分で懸念された薄板曲げのせん断ロッキングを避ける。

| 項目 | 値 |
|---|---:|
| 要素数 | 560 |
| 節点数 | 3,357 |
| 総自由度 | 10,071（拘束前） |
| 板厚方向 | 1 mm × 2要素 |
| X方向要素寸法 | 10～25 mm |
| Y方向要素寸法 | 5～25 mm |
| 固定節点 | 349節点 × 3自由度 |

一様格子ではなく、次の箇所を局所細分化する。

- 固定領域の内部と境界 `x=200/300 mm, y=0/20 mm`
- 中央センサ `(200,100) mm`
- 8打点 `x=50/150/250/350, y=50/150 mm`
- 固定領域直下の曲率変化が大きい範囲

外側は最大25 mmまで粗くして計算量を抑える。完全非構造メッシュではなく、要素品質と打点・センサの節点一致を優先したgraded hexahedral meshである。

## 材料・境界条件

単位系はmm、N、s、tonne。

| 条件 | 値 |
|---|---:|
| ヤング率 | 3,200 MPa |
| ポアソン比 | 0.35 |
| 密度 | 1.18×10⁻⁹ tonne/mm³ = 1,180 kg/m³ |
| 形状 | 400 × 200 × 2 mm |
| 固定体積 | x=200–300、y=0–20、z=-1–1 mm |
| センサ | 上面 `(200,100,1) mm`、1点のみ |

固定体積に含まれる全節点のX・Y・Z変位を0とする。接触、摩擦、締付け圧はまだモデル化せず、完全固定としている。

## 固有値解析

```text
*FREQUENCY,STORAGE=YES,SOLVER=SPOOLES
30,0.,1200.
```

30モードを要求し、固有値、質量・剛性行列、質量正規化モードをEIGへ保存する。先頭12固有振動数は次のとおり。

```text
8.9348, 14.7523, 23.3173, 38.9897,
57.3940, 77.1832, 95.8313, 104.3011,
116.2387, 138.8426, 150.0794, 194.8130 Hz
```

独自一次HEX8より低い結果は過剰剛性の疑いと整合するが、モード番号だけでは形状対応を保証できない。MACで対応付け、メッシュ収束、実測FRFで確認するまで確定値としない。

## 打撃とモード動解析

各打点を独立ジョブとしてCalculiXの`*MODAL DYNAMIC,DIRECT`で計算する。固有値解析で保存したEIGを各ジョブへコピーし、CalculiX内部の質量正規化モードと整合質量行列を使用する。

| 条件 | 値 |
|---|---:|
| 時間刻み | 0.15625 ms = 1/6,400 s |
| 計算時間 | 320 ms |
| 出力点数 | 2,048 |
| 荷重 | 2時間刻み幅の三角形Z方向力積 |
| 三角形ピーク | 6,400 N、面積1 N·s |
| 使用モード | 30 |
| モード減衰比 | 1.2%、Mode 1–30 |
| 出力 | 中央センサ節点のZ変位 |

中央変位を同一時間刻みで2階数値微分して加速度へ変換する。FFTにはHann窓を用い、分解能は3.125 Hz。時間波形とFFTはそれぞれ8打点共通最大値で正規化する。

## 再現

```powershell
.\run-calculix.ps1
```

生成物には、CalculiX入力デッキ`.inp`、固有値・質量・剛性を格納する`.eig`、モード形状`.frd`、数値結果`.dat`、各ジョブの実行ログ、SVG、JSON、JavaScriptを含む。

`calculix-eight-hits.mp4`は直上ビュー、`calculix-eight-hits-perspective.mp4`は変形を立体的に読むための斜めパースビューである。どちらもCalculiXがFRDへ出力した30個の質量正規化モードから、8打点の60 ms過渡表面変位を再構成する。24 fps、1280 × 650、H.264/yuv420p、5秒のスローモーションである。8打点は共通振幅スケールを使い、パースのZ方向は正規化最大変位を±18 mmとして意図的に強調する。この高さは実寸変形ではない。中央センサの時間波形は別途CalculiXの`*MODAL DYNAMIC`結果と照合している。

## 残る検証

1. 粗・標準・細の3メッシュで固有振動数とMACの収束を確認する。
2. 板厚方向1・2・4要素を比較する。
3. C3D20Rの低減積分による異常モードがないことを目視・ひずみエネルギーで確認する。
4. センサ基板と接着層の質量・剛性を追加する。
5. 完全固定を接触・摩擦・締付け圧へ置換する。
6. 実測FRFとMACで材料定数・減衰・固定剛性を同定する。
