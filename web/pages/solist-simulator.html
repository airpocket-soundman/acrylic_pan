08 / Solist AI Simulator

512点の打撃波形を、
8エリア識別モデルへ変換する。

IchiPing_solist_AIで確立した公式Simulatorの使い方を、acrylic_panの振動位置推定へ移植します。収録した波形をそのまま学習器へ渡すのではなく、同じ前処理で固定長の特徴量へ変換し、8出力のone-hot教師データで学習します。

このPCで確認済みの実行環境公式 Solist-AI Sim 教師あり学習版 SLV1.00.04 がインストールされています。実行ファイルは次の場所です。
C:\Program Files\ROHM\SolistAI_Sim_SLV10004sp\application\SolistAI_Sim_SLV10004.exe

管理者権限は通常不要です。学習データとモデル成果物はProgram Files配下ではなく、このリポジトリの作業ディレクトリへ保存します。

acrylic_pan用の入出力

CAPTURE

512点のZ軸波形

25.6 kHz、20 ms。リングバッファからプリトリガ128点＋ポストトリガ384点を切り出します。

FEATURES

128 FFT特徴

DC除去とHann窓の後、DCを除く先頭128ビンのlog振幅を計算し、学習データだけから求めた平均・標準偏差で正規化します。

→
OUTPUT

8クラススコア

各エリアをone-hotの8値で学習。実機では8スコアの最大値をCPU側で選び、クラス番号へ変換します。

Simulatorへ渡すCSV

1行が1打撃です。Simulatorが読み込むCSVには識別用文字列や日時を混ぜず、数値だけを並べます。収録メタデータは別ファイルで保持し、CSV生成時に結合します。

input_0,input_1,...,input_127,target_0,...,target_7
-0.184,0.927,...,0.311,1,0,0,0,0,0,0,0
 0.442,-0.105,...,1.028,0,1,0,0,0,0,0,0
入力データFirst column = 1Rows at one chunk = 1、Columns at one chunk = 128
期待値First column = 129Rows = 1、Columns = 8
チャンク数AutomaticCheck後に入力128／出力8ノードになることを確認
ヘッダ1行目に列名Simulatorは先頭ヘッダを読み飛ばす前提。読込み結果も必ず確認
分割収録セッション単位同じ日の連続打撃をtrainとtestへランダム混在させない
サイズ制約（データ行数＋ヘッダ）× 136列 ≤ 1,000,000安全側で最大7,351データ行。生成コードが超過を拒否

初期AI設定

ACTIVATION

Hard sigmoid

Solist AIのELM系1隠れ層で使用する非線形写像。PC側の比較コードも同じ定義にします。

LOSS

MSE

8クラスのone-hot教師値と8出力スコアの二乗誤差を最小化します。

PRECISION

double → bfloat16

最初はdoubleで構成とSeedを探索し、候補を決めてからbfloat16を有効にして実機相当精度を確認します。

RANDOM SEED

Seed 1を基準

IchiPingの再現設定を初期値にし、1～10を同じtestデータで比較。採用Seedはモデルversionへ固定記録します。

Seed1（基準）1～10比較後、採用値を固定

作業フロー

01

実測データを収録

8エリアを同数、打撃強度と日・固定し直しを変えて収録します。生の512点波形とラベル、capture tokenを失わず保存します。

02

特徴量CSVを生成

波形品質を検査し、128 FFT特徴へ変換。trainで正規化定数を作り、validation/testにも同じ定数を適用します。

03

doubleで候補を探索

隠れ32／64／128、Seed 1～10を比較します。総合精度だけでなく8×8混同行列と最悪クラス精度を確認します。

04

bfloat16で再評価

有力候補を実機精度へ切り替え、精度低下、AI RAM、処理時間を記録します。別セッションtestで8エリア90%以上、デモ目標95%以上を合否基準にします。

05

モデルを保存してファームへ組込み

採用モデルをSimulatorから保存し、model1.hを推論ファームへ取り込みます。CPU側でargmax、最低スコア、1位と2位のmarginを判定します。

06

PCと実機の出力を照合

保存済み波形を使い、PC前処理、Simulator、実機の8スコアを比較。差が許容範囲内であることをリリース条件にします。

保存するモデル成果物

model1.xlsxSimulator再ロード用設定・係数を再現し、後から評価をやり直すため保存
model1.mat数値解析用Simulatorが出力した元データとして保持

metrics.json / confusion.csv採用根拠Seed、隠れ数、精度モード、クラス別精度、AI RAM、処理時間を記録
推奨ディレクトリmodels/area8/<model-version>/ に上記一式をまとめます。ファイル名だけでなくmanifest内にも収録データversion、特徴量version、Seed、Simulator SLV1.00.04を記録します。

XY座標＋強度へ進む条件

まず8クラスモデルを基準性能として完成させます。座標回帰へ進む場合も同じ128特徴を入口にできますが、教師値を8 one-hotから正規化した x, y, strength の3列へ変更し、クラス分類とは別モデルとして評価します。8エリアの中心以外を含む十分な座標データが集まるまでは、クラス出力からエリア中心座標へ変換する方式を採用します。

このページの設定はIchiPing_solist_AIで使用した公式Sim運用を基準に、acrylic_panの8打点・512点振動波形向けへ改変した初期仕様です。帯域境界、隠れノード数、Seedは実測データで最終決定します。
