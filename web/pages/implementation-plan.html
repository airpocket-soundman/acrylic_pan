07 / Implementation roadmap

波形を確実に取るところから、
8クラス・XY推定へ育てる。

最初の合格点は、衝撃の立ち上がりを欠落させずに1イベント分の振動波形を取得し、PCで時間波形とFFTを確認できることです。その同じ通信・収録基盤へ、ラベリング、学習、推論、演奏表示を順番に追加します。

現在のマイルストーン — M1PCモニタの通信プロトコル、波形・FFT表示と保存処理を実装中。次にML63Q2557/KX134側のリングバッファ、衝撃検出、UART転送を接続します。

実装する順番

M0

共通仕様と自動テスト

サンプリング条件、COBS + CRC32通信、イベント形式、sequence番号を固定します。PC上の疑似イベントで分割受信、CRC異常、FFTピークをテストします。

M1

衝撃検知・波形・FFTモニタ

マイコンはKX134のZ軸を常時取得し、jerkしきい値で打撃を検知。プリトリガを含む512点をPCへ送り、PCは時間波形とFFTを表示してNPZへ自動保存します。

M2

教師データ収録とラベリング

PC画面に次に叩く8エリアを表示し、受信した波形へarea ID、座標、打撃順、品質フラグを付与。取り直しと収録数の均等化も行います。

M3

特徴量評価とSolist AI用データ生成

生波形、FFT、帯域エネルギーなどを比較し、実測データを学習・検証・テストへ打撃単位で分割。正規化定数も学習データだけから決めます。

M4

8クラス推論

Solist AI Simulatorで8出力スコアのモデルを生成してファームへ組み込みます。モデルは8スコアを返し、CPU側で最大値としきい値からクラスを決定します。

M5

演奏モード

推論結果をPCへ送り、対応音階を再生。アクリル板上の打点エリア、スコア、強度、レイテンシをリアルタイム表示します。

M6

XY座標＋強度への拡張

実測量と精度を確認後、出力をx、y、strengthの3値回帰、または8クラス＋座標補正へ拡張します。8クラス方式を基準性能として残します。

M7

耐久・再現性・リリース

連打、弱打、強打、温度・固定条件の変化、長時間通信を評価。モデル・ファーム・収録条件へversionを付け、再現できる一式にまとめます。

M1のバッファ設計

CONTINUOUS

リングへ常時記録

25.6 kHzでZ軸を連続取得。書込み位置は循環し、最新の過去波形を常に保持します。

→
TRIGGER

衝撃を検出

サンプル間の差分（jerk）がしきい値を超えた位置をtrigger indexとします。

→
POST CAPTURE

後半を待って確定

過去128点をコピーし、以後384点を追記。512点が揃ってから送信します。

イベント長512 samples / 20 msSolist AIサンプルの入力長と一致
プリトリガ128 samples / 5 ms衝撃の立ち上がり前とDC基準を確保
ポストトリガ384 samples / 15 ms主要な減衰振動を取得
サンプルレート25,600 samples/sKX134の既存サンプル設定を使用
欠落対策リング＋イベント領域検知後に取得を始める方式ではないため先頭を失わない
検出境界前ブロック末尾も保持DMA/コールバック境界を跨ぐ差分も正しく判定
UART送信待ちの扱い512点イベントはヘッダ込みで約1.1 kBとなり、115200 bpsでは送信に約95 msかかります。初版は1イベントを確定して送信し、送信完了後に再アームします。したがって最初の打撃の先頭は欠落しませんが、約100 ms以内の連打対応にはイベント領域の2面化または高ボーレート化が必要です。

M1の合格条件

  • 静止時に誤トリガせず、しきい値をPCから調整できる
  • 全イベントでtrigger indexが128となり、その前に128点の実測波形が存在する
  • 1 Hzで100回叩き、sequence欠落・CRCエラー・overrunが0
  • PCで時間波形と周波数スペクトルが同時表示される
  • 受信イベントがtimestamp、sample rate、trigger index付きで自動保存される
  • 疑似データで通信フレームとFFTの自動テストが通る

コマンドラインでのビルドと書込み

ビルド確認済みLEXIDE付属のmakeとLLVMツールチェーンでDebug/makefileを実行可能。出力はELFとHEXです。
書込みツール確定待ち付属資料ではDAPLink + KeilまたはJ-Link + J-Flashを使用。どちらもCLI自動化できますが、このPCでは書込み用CLI本体をまだ検出できていません。
今後のスクリプトbuild-firmware.ps1環境変数とmakeを固定し、クリーン／増分ビルドを再現可能にします。
今後のスクリプトflash-firmware.ps1実際に接続するDAPLink/J-LinkとCLIの場所を確認後、HEX書込みと結果確認を自動化します。

現在作成済みのPC側部品

PROTOCOL

APAN通信

COBS、CRC32、sequence、EVENT_DATA、コマンド応答を実装しています。

MONITOR

波形＋FFT GUI

COM接続、時間波形、FFT、イベント情報、NPZ自動保存、デモ入力を実装しています。

SOLIST DATA

8クラスCSV生成

512点波形を128 FFT特徴へ変換し、学習データだけで標準化してone-hot教師値付きCSVを出力します。

REFERENCE ELM

互換モデル検査

固定alpha・学習betaの純NumPy ELMで、公式Simulator投入前の次元、精度、将来のXY＋強度回帰を検査します。

数値は初期値です。実機波形を見ながらtrigger threshold、プリトリガ長、再アーム時間を変更できるようにします。
