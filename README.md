# Acrylic Pan

ROHM Edge Hack Challenge 向けに、400 × 200 × 2 mm のアクリルパネルを
打楽器兼タッチインターフェースにするプロジェクトです。

DT-EBML63Q2557 に接続した KX134-1211 加速度センサ1個をパネル中央へ取り付け、
打撃位置による振動モード強度の違いを測定します。Solist-AI は4 × 2領域の
空間スコアを出力し、音階、確率ヒートマップ、期待座標 `(x, y)` へ変換します。

## 目標

- 第一目標: 4 × 2（8領域）の安定識別
- AI出力: 8領域のスコア分布。確率期待値から連続座標 `(x, y)` も算出
- 座標精度の評価: 平均絶対誤差、距離誤差、95パーセンタイル誤差
- 演奏出力: 推定領域ごとに異なるスチールパン風音色
- 表示: パネル上の推定座標と履歴ヒートマップ
- 複数打撃: 同時2点はmulti-hot出力、100 ms以上離れた連打は別イベントとして識別
- 発展目標: 実測誤差が十分小さければ50 mmまたはそれ以下へ細分化

## 座標系

パネルを正面から見て左上を原点とします。

```text
(0, 0) ┌──────────── 400 mm ────────────┐
       │  C4  │  D4  │  E4  │  G4  │
       ├──────┼──────┼──────┼──────┤ 200 mm
       │  A4  │  C5  │  D5  │  E5  │
       └──────────────────────────────┘ (400, 200)
```

Solist-AI の教師値は次の0〜1正規化座標です。

```text
x_norm = x_mm / 400
y_norm = y_mm / 200
```

推論後の100 mm領域は `column = floor(x_mm / 100)`、
`row = floor(y_mm / 100)` で求めます。範囲外の推論値はパネル端へ制限します。

## 信号処理の初期案

1. KX134-1211のZ軸加速度を最大ODR 25.6 kHzで連続取得
2. 加速度差分で打撃開始を検出
3. リングバッファからプリトリガ128点を含む512サンプル（20 ms）を切り出す
4. DC成分除去後、整列した時間波形とFFT／帯域強度を比較
5. Solist-AIの教師ありモデルから8領域のスコアを出力
6. 有効な上位1～2スコアを音階へ、全スコアをヒートマップと期待座標へ変換

512点は収録イベント長であり、その全点を必ずAI入力にするわけではありません。
時間波形の間引き、FFT帯域、帯域強度要約を実測で比較し、570ノードの目安と
精度の両方を満たす入力次元を決定します。出力は8クラスのone-hot教師値です。

## Solist-AI Simulator

このPCでは公式の教師あり版 Solist-AI Sim SLV1.00.04 と MATLAB Runtime R2024aを使用します。
インストール状態の確認と起動は次のスクリプトで行えます。

```powershell
.\scripts\launch-solist-ai-sim.ps1 -CheckOnly
.\scripts\launch-solist-ai-sim.ps1
```

公式Simulatorへ入力するCSVの生成条件、8クラス用の列設定、モデル保存方法は
[Simulator実行環境](docs/solist-ai-simulator.md)にまとめます。Python側にもSolist-AI互換の
ELM参照実装を置き、公式Simulatorへ渡す前に次元・精度・データ分割を自動検査します。

## データ収集

教師中心は X = 50, 150, 250, 350 mm、Y = 50, 150 mmです。中心だけでなく
各100 mm領域内へ打撃点を散らし、強弱・打具・打撃者・時間を変えて収録します。
同じ収録セッションを学習とテストに分割せず、別セッションで評価します。

## 開発段階

1. PCへ加速度データを送り、打撃波形と教師座標を収録
2. Dockerの薄板モデルで固有モードと中央センサへの結合強度を解析
3. Solist-AI Sim教師あり版で8出力・隠れ層1層モデルを評価
4. DT-EBML63Q2557上で推論し、座標をUART出力
5. PCでスチールパン音源とヒートマップを実装
6. 必要に応じてPWM音源などへ移し、スタンドアロン化
